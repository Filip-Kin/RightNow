# RightNow Wear OS companion

Standalone Wear OS (Pixel Watch 3) app for answering the hourly quick-log prompt from
the wrist. Companion-only: the phone drives timing and owns encryption/sync; the watch
just produces plaintext answers over the Wearable Data Layer.

This is a **separate Gradle project** from the Expo app. It is committed (the Expo
`app/android/` tree is gitignored + regenerated by prebuild, so a module there would be
clobbered). It shares no code with React Native.

## How it fits together

- Phone -> watch (DataClient, fixed paths): `/rightnow/taxonomy`, `/rightnow/reminder`
  (cached by `WearListenerService` for the UI), and `/rightnow/prompt` each hour (the
  watch posts its own local notification on this).
- Watch -> phone (DataClient, unique path `/rightnow/answer/<ts>-<hour>`): the phone's
  `RightNowWearListenerService` appends each answer to `quicklog-queue.json`, deletes the
  consumed item, and kicks the existing `RightNowQuickLogDrain` headless task, which
  encrypts (E2EE) and pushes to the server. No app launch needed; survives the phone
  being briefly out of range.

Pairing requires the watch app to share the phone's `applicationId`
(`com.filipkin.rightnow`) and signing certificate (the committed `app/ci/debug.keystore`).

## Build

Pinned toolchain: AGP 8.7.3, Gradle 8.11.1, Kotlin 2.0.21, compileSdk 35 (all covered by
`Dockerfile.android`'s installed SDK + JDK 17). CI builds it via the Gradle installed in
that image:

```sh
gradle -p wear :app:assembleRelease   # output: wear/app/build/outputs/apk/release/app-release.apk
```

Locally (Android Studio handles the wrapper automatically), or from the CLI generate the
wrapper once with a matching Gradle, then:

```sh
cd wear && gradle wrapper --gradle-version 8.11.1 && ./gradlew :app:assembleRelease
```

## Install on the watch (wireless ADB)

1. On the watch: Settings -> Developer options -> enable ADB debugging + Wireless debugging.
2. `adb connect <watch-ip>:<port>` (pair first if prompted), then
   `adb -s <watch-ip>:<port> install -r RightNow-wear.apk`.
3. Open the phone RightNow app once so it pushes the current taxonomy + reminder state to
   the watch (the watch UI needs the taxonomy to render the activity grid).
